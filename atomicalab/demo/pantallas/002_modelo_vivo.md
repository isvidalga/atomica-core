## Estado

Congelado v1.0

---

# Pregunta

¿Cómo se relacionan las partes de mi organización?

Esta pantalla no responde:

- cómo está la empresa
- qué ocurrió
- dónde intervenir
- qué intervenciones existen
- qué evidencia soporta el modelo

---

# Objetivo

Permitir comprender la estructura sistémica de la organización.

No muestra resultados.

Muestra relaciones.

---

# Usuario

CEO.

Uso previsto:

- cuando necesita comprender un problema

- cuando una dimensión cambia

- antes de decidir una intervención

---

# Arquitectura

Existe un único elemento protagonista.

La red organizacional.

Todo lo demás es apoyo.

---

# Elementos

## 1. Grafo

Representa únicamente relaciones.

No representa rendimiento.

No representa puntuaciones.

No representa tiempo.

Cada nodo corresponde a un sistema organizacional.

Las conexiones representan influencia.

---

## 2. Sistema seleccionado

Cuando un nodo es seleccionado aparecen:

Nombre

Descripción

Capacidades

Fragilidades relacionadas

No aparecen gráficos.

No aparecen tablas.

---

## 3. Relaciones

Cada relación muestra únicamente:

Origen

Destino

Intensidad

Sentido

No calcula causalidad.

Representa influencia estimada.

---

## 4. Capacidad crítica

Siempre existe una capacidad destacada.

Es la capacidad cuya mejora produciría mayor impacto sobre el sistema.

Puede abrir Donella.

# Principios

El modelo representa relaciones.

No representa hechos.

No representa verdad.

Representa hipótesis organizacionales.

Toda relación posee incertidumbre.

Toda relación puede cambiar.

---

# Criterio de aceptación

Después de recorrer el Modelo Vivo un CEO debe responder:

¿Qué sistemas están relacionados?

¿Qué capacidad conecta más elementos?

¿Dónde puede propagarse un problema?

¿Qué parte del sistema merece ser comprendida antes de intervenir?
