# Grafo de Relaciones

## Propósito

El Grafo de Relaciones representa la estructura interna de la organización.

No muestra organigramas.

No muestra departamentos.

No muestra jerarquías.

Representa dependencias.

Influencia.

Propagación.

Conocimiento.

El objetivo es comprender cómo se relaciona el sistema consigo mismo.

---

# Principio

Nada existe de forma aislada.

Toda decisión afecta.

Todo conocimiento sostiene.

Todo patrón modifica.

Toda evidencia conecta.

ATÓMICA representa esas conexiones.

---

# Nodos

Todo elemento de ATÓMICA puede convertirse en un nodo.

Organización.

Persona.

Rol.

Proceso.

Decisión.

Patrón.

Saber.

Hipótesis.

Acción.

Evidencia.

Riesgo.

Indicador.

Diagnóstico.

Proveedor.

Cliente.

Proyecto.

---

# Relaciones

Toda relación posee:

origen

destino

tipo

peso

confianza

fecha

estado

Nunca existe una relación sin significado.

---

# Tipos

Los tipos mínimos son:

depende_de

explica

provoca

mitiga

refuerza

contradice

genera

necesita

verifica

aprende_de

participa_en

impacta

observa

protege

sustituye

---

# Dirección

Todas las relaciones poseen dirección.

La dirección forma parte del conocimiento.

No todas son simétricas.

---

# Peso

El peso representa intensidad.

No frecuencia.

No cantidad.

Una relación muy poco frecuente puede poseer enorme peso sistémico.

---

# Confianza

Toda relación mantiene una confianza.

Alta.

Media.

Baja.

Hipotética.

Una relación puede existir antes de quedar demostrada.

---

# Tiempo

Las relaciones evolucionan.

Pueden:

nacer

fortalecerse

debilitarse

desaparecer

reactivarse

El tiempo modifica el grafo.

---

# Clusters

Los nodos generan comunidades.

Las comunidades aparecen.

Nunca se diseñan manualmente.

Representan estructuras reales.

No categorías.

---

# Centralidad

ATÓMICA calcula:

grado

intermediación

cercanía

vector propio

El objetivo no es académico.

El objetivo es descubrir cuellos de botella.

---

# Propagación

Todo cambio inicia una propagación.

Una evidencia modifica un Saber.

El Saber modifica un Patrón.

El Patrón modifica un Riesgo.

El Riesgo modifica una Prioridad.

La propagación puede recorrerse.

---

# Dependencias

ATÓMICA identifica:

dependencias fuertes

dependencias débiles

dependencias críticas

dependencias ocultas

Las dependencias constituyen uno de los principales indicadores de fragilidad.

---

# Explicabilidad

Toda conclusión debe poder recorrerse.

El usuario puede preguntar:

¿Por qué existe este patrón?

↓

¿Qué conocimiento lo sostiene?

↓

¿Qué evidencias lo demuestran?

↓

¿Quién las aportó?

↓

¿Cuándo?

↓

¿Qué decisiones generaron?

El recorrido nunca se rompe.

---

# Navegación

La navegación siempre comienza en un nodo.

Nunca en el grafo completo.

El usuario explora.

Nunca observa un mapa inmóvil.

---

# Visualización

El grafo es un instrumento.

No una ilustración.

La posición responde al equilibrio de relaciones.

No a criterios estéticos.

---

# Filosofía

Las organizaciones creen que trabajan mediante procesos.

En realidad trabajan mediante relaciones.

ATÓMICA hace visibles esas relaciones para comprender cómo fluye el conocimiento, cómo se propagan los riesgos y dónde nacen las decisiones.
